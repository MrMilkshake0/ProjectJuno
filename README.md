# 🌙 Project Juno — Beyond the Swipe

**Website:** [https://www.projectjuno.ai](https://www.projectjuno.ai)

Project Juno is an experiment in building the **next generation of dating systems** — ones that prioritize **compatibility and outcomes** over engagement and addiction.

It’s not about swiping or optimizing for screen time.  
It’s about **understanding people** — their values, habits, communication styles, and rhythms — and using that data to improve the odds of meaningful connection.

---

## 💡 What It Is

> “You can’t guarantee chemistry — but you can improve the odds of it happening.”

Project Juno explores how **data, psychology, and machine learning** can model interpersonal alignment more thoughtfully.  
It aims to create an outcome-focused ecosystem — a system that connects people based on **compatibility vectors** rather than attention loops.

The project currently includes:
- A working **React + TypeScript** web app (`projectjuno.ai`) built with [Vite](https://vitejs.dev/), [Tailwind](https://tailwindcss.com/), and [Shadcn/UI](https://ui.shadcn.com/).
- A **structured questionnaire prototype** that captures ~100 user data points (values, temperament, lifestyle, and relationship expectations).
- A public **whitepaper** outlining the conceptual and ethical framework behind Juno.

---

## 🧭 Core Philosophy

| Focus | Description |
|--------|--------------|
| **Outcome-focused** | Matching systems that optimize for long-term connection, not clicks. |
| **Human-in-the-loop** | Every match involves a human check before introduction. |
| **Transparent by design** | Explainable matching logic and clear data usage. |
| **Private by default** | Drafts stay local; submissions are securely stored and deletable anytime. |
| **Fairness-first** | No demographic weighting or engagement bias. |

## 🧩 Key Components

### 1. Front Page (`/src/pages/FrontPage.tsx`)
A minimal, responsive landing page introducing Project Juno.  
Features SEO metadata, structured JSON-LD (Organization & WebSite schemas), and a clear CTA to start the questionnaire or learn more.

### 2. About Page (`/src/pages/AboutPage.tsx`)
Explains Juno’s philosophy, roadmap, and vision.  
Structured into cards for easy reading with an “On This Page” sidebar for navigation.

### 3. Whitepaper (`/src/pages/WhitepaperPage.tsx`)
A long-form document that details:
- The motivation behind Project Juno  
- Conceptual framework and ethical foundations  
- Roadmap toward a compatibility-driven matching system  
- Future development paths

The page includes a responsive **table of contents**, **scroll-spy navigation**, and **semantic typography components** (`H1`, `H2`, `BQ`, etc.) for consistency.

---

## 🔒 Ethics & Data Principles

- **Consent-first:** participation is opt-in only.  
- **Data ownership:** users can request deletion anytime.  
- **Fairness:** no demographic or engagement-based weighting.  
- **Transparency:** clear explanations for recommendations.  
- **Minimal collection:** only what’s needed to model compatibility.
